# Dataset Rule Notes

This file is documentation only. It is not loaded by the code.

The rule names are shared across datasets, but their meanings follow each dataset's
field mapping. The lists below spell out the full rules used for each dataset.

## Fodors-Zagats (FZ)

Graph pattern:

```text
Restaurant -> Address -> City
```

Field mapping:

```text
name    = restaurant name
address = address node value
city    = city node value
phone   = restaurant phone
cuisine = restaurant cuisine
```

Full rule set:

```text
phone_exact
  Normalized restaurant phone numbers are identical.

phone_area
  Restaurant phone area codes are identical.

name_sim_ge_045
  Restaurant-name token Jaccard similarity >= 0.45.

name_sim_ge_060
  Restaurant-name token Jaccard similarity >= 0.60.

name_sim_ge_075
  Restaurant-name token Jaccard similarity >= 0.75.

address_sim_ge_030
  Address token Jaccard similarity >= 0.30.

address_sim_ge_050
  Address token Jaccard similarity >= 0.50.

cuisine_sim_ge_050
  Cuisine token Jaccard similarity >= 0.50.

same_city
  City strings are identical after normalization.

name045_address030
  Restaurant-name similarity >= 0.45 and address similarity >= 0.30.

name045_phone_area
  Restaurant-name similarity >= 0.45 and phone area codes match.

phone_exact_or_name040_phone_area
  Phone exact, or phone area matches and restaurant-name similarity >= 0.40.

phone_exact_or_name035_phone_area
  Phone exact, or phone area matches and restaurant-name similarity >= 0.35.

address050_phone_area
  Address similarity >= 0.50 and phone area codes match.

name_or_cuisine_or_char060
  Restaurant names have token overlap, or cuisine similarity >= 0.50,
  or character-level restaurant-name similarity >= 0.60.

address030_or_exact_name_or_name075
  Address similarity >= 0.30, or normalized restaurant names are identical,
  or restaurant-name similarity >= 0.75.

phone_name_candidate_consistent
  Phone/name consistency and name/cuisine/character consistency and
  address/name consistency all hold.
```

## DBLP-ACM (DA)

Graph pattern:

```text
Paper -> Author
Paper -> Venue
```

Field mapping:

```text
name    = paper title
address = author list
city    = venue
phone   = publication year
cuisine = venue
```

Full rule set:

```text
phone_exact
  Publication years are identical.

phone_area
  Publication-year prefixes match under the generic numeric rule.

name_sim_ge_045
  Paper-title token Jaccard similarity >= 0.45.

name_sim_ge_060
  Paper-title token Jaccard similarity >= 0.60.

name_sim_ge_075
  Paper-title token Jaccard similarity >= 0.75.

address_sim_ge_030
  Author-list token Jaccard similarity >= 0.30.

address_sim_ge_050
  Author-list token Jaccard similarity >= 0.50.

cuisine_sim_ge_050
  Venue token Jaccard similarity >= 0.50.

same_city
  Venue strings are identical after normalization.

name045_address030
  Paper-title similarity >= 0.45 and author-list similarity >= 0.30.

name045_phone_area
  Paper-title similarity >= 0.45 and publication-year prefix matches.

phone_exact_or_name040_phone_area
  Publication year exact, or year prefix matches and title similarity >= 0.40.

phone_exact_or_name035_phone_area
  Publication year exact, or year prefix matches and title similarity >= 0.35.

address050_phone_area
  Author-list similarity >= 0.50 and publication-year prefix matches.

name_or_cuisine_or_char060
  Paper titles have token overlap, or venue similarity >= 0.50,
  or character-level title similarity >= 0.60.

address030_or_exact_name_or_name075
  Author-list similarity >= 0.30, or normalized paper titles are identical,
  or title similarity >= 0.75.

phone_name_candidate_consistent
  Year/title consistency and title/venue/character consistency and
  author/title consistency all hold.
```

Rules selected in the DA run:

```text
Stage1 selected rules:
  name_or_cuisine_or_char060
  phone_area
  phone_exact_or_name035_phone_area

Stage2 acceptance:
  confidence >= 0.95
```
